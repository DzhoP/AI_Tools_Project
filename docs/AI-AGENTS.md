# AI агенти в проекта

Този документ описва как проектът се разработва с помощта на AI агенти и как нова агент сесия може да продължи работата.

## Използвани инструменти

| Инструмент | Роля в проекта |
|---|---|
| **Claude Code CLI** (VS Code разширение) | Основният агент за разработка — писане на код, дебъгване, Docker операции, обяснения |
| Модели: Claude Fable 5 / Sonnet | Fable за по-сложни задачи, Sonnet за ежедневна работа |

## Как работим с агента

Процесът, който се утвърди в проекта:

1. **Задачата се обсъжда преди код** — агентът задава уточняващи въпроси (обхват, права, поведение при гранични случаи) и чак след отговорите пише код. Ключовите решения ги взима човекът.
2. **Агентът пише и копира** — заради Docker sync проблема всяка редакция се последва от `docker cp` в контейнера. Агентът прави това автоматично.
3. **Тестване на две нива** — агентът тества API-то с curl директно, а човекът проверява визуално в браузъра и докладва какво вижда.
4. **Грешките се дебъгват съвместно** — човекът поставя текста на грешката от браузъра, агентът проверява логове и код.

## Контекст за агента

- **`CLAUDE.md`** (в корена) — чете се автоматично от Claude Code при стартиране в тази папка. Съдържа стека, критичния Docker sync проблем, домейн правилата и конвенциите.
- **`docs/PROMPTS.md`** — готови начални промптове за стартиране на нова агент сесия по типове задачи.

## Научени уроци от работата с AI агент

- Дългите чат сесии стават скъпи и бавни — по-добре нова сесия с добър контекстов файл, отколкото безкраен чат.
- Агентът предлага, но решенията за архитектура, права и обхват трябва да са на човека — иначе не можеш да защитиш проекта си.
- Всичко генерирано трябва да се разбира — правилото в академията е, че AI кодът, който не можеш да обясниш, е дълг, не актив.
